# ROS 机器人项目审查清单

## 一、工作区与构建

- 是否先识别了 ROS 1 / ROS 2 / 混合工作区
- 是否沿用了现有构建系统，而不是盲目切换 `catkin_make` / `catkin build` / `colcon`
- `package.xml` 与 `CMakeLists.txt` / `setup.py` 是否一致
- `rosdep` 依赖是否齐全
- 目标文件、脚本、launch、配置、URDF 是否正确安装/导出

## 二、消息与接口

- topic / service / action 名称是否稳定且语义清晰
- 自定义消息是否存在单位不一致、字段语义漂移
- namespace 与 remap 是否显式、可追踪
- ROS 2 是否处理了 QoS；ROS 1 是否确认了 source 环境

## 三、TF 与时间

- TF 树是否单根
- 静态 TF 与动态 TF 是否放对位置
- `frame_id` / `child_frame_id` 是否一致
- 时间戳是否连续、来源是否可信
- 仿真是否正确使用 `/use_sim_time`

## 四、传感器与底盘

- IMU 坐标轴、重力方向、角速度单位是否一致
- LiDAR/相机驱动是否校验了权限、串口/USB、时戳与 frame
- 底盘速度、轮距、编码器方向、里程积分是否清楚
- 失连保护、急停、超时停车是否存在

## 五、导航与控制

- 代价地图、控制频率、局部/全局规划参数是否匹配机器人能力
- 定位源是否稳定，里程计是否足够可信
- 控制链路中是否存在高延迟 topic、过低频率或错误坐标系
- 是否把“导航跑不动”误判为算法问题，而真实原因是 TF/时间/驱动

## 六、嵌入式与总线

- 串口/CAN 帧格式、字节序、校验、超时是否明确
- 是否存在 ISR 阻塞、动态分配、无界循环、竞争访问
- 板端和主机是否约定统一时间源、采样频率和状态机
- DMA/缓存/对齐/共享缓冲区是否有竞态或一致性问题

## 七、回归验证

- 最小构建是否通过
- 目标包测试是否通过
- 启动文件是否能在干净环境跑起
- 关键 topic 频率、TF、参数、执行器链路是否回归正常
- 是否给出现场可执行的排障命令与验证步骤
