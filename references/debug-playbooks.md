# ROS 现场排障 Playbooks

## 总原则

不要一上来改代码，先走证据链。

## Playbook 1：能编译但节点起不来

1. 是否 source 了正确环境
2. launch / launch.py 路径是否正确
3. 资源文件是否被正确安装
4. Python 节点入口、权限、依赖是否正确
5. 参数 YAML 是否能被加载

## Playbook 2：topic 存在，但系统表现不对

1. topic 名称是否对
2. 发布频率是否合理
3. 时间戳是否连续
4. `frame_id` 是否正确
5. 单位是否统一
6. QoS 是否匹配（ROS 2）

## Playbook 3：MCU 联调时偶发异常

1. 总线吞吐是否足够
2. 时间戳来自哪里
3. 丢包与重传是否可观测
4. 主机侧是否单线程做了耗时解析
5. 板端是否存在阻塞、动态分配、无界重试
